# Pre-Deployment Checklist

Complete this checklist before deploying to production to ensure everything is ready.

## üìã Pre-Deployment Tasks

### Code Quality

- [ ] All TypeScript errors resolved (or intentionally ignored)
- [ ] ESLint warnings reviewed
- [ ] No console.log statements in production code (auto-removed by build)
- [ ] All TODO comments addressed or documented
- [ ] Code committed and pushed to GitHub

### Content Verification

- [ ] All placeholder content replaced with real content
- [ ] About page has authentic bio and information
- [ ] At least 3 projects created in Sanity
- [ ] At least 1 blog post published
- [ ] All images have proper alt text
- [ ] Profile image uploaded and optimized
- [ ] Social links are correct and working

### Sanity CMS

- [ ] Sanity project created at sanity.io
- [ ] Project ID and dataset name noted
- [ ] Studio accessible at /studio route locally
- [ ] All schemas working correctly
- [ ] Test content created and published
- [ ] Image uploads working
- [ ] GROQ queries tested in Vision tool

### Environment Configuration

- [ ] `.env.local` file created with all variables
- [ ] `NEXT_PUBLIC_SANITY_PROJECT_ID` set correctly
- [ ] `NEXT_PUBLIC_SANITY_DATASET` set to production
- [ ] `NEXT_PUBLIC_SITE_URL` set to https://kazekeza.com
- [ ] `.env.local` is in `.gitignore` (never commit secrets)

### Local Testing

- [ ] Site builds successfully (`npm run build`)
- [ ] Production build runs locally (`npm run start`)
- [ ] All pages load without errors
- [ ] Content from Sanity displays correctly
- [ ] Images load and are optimized
- [ ] Theme toggle works
- [ ] Mobile responsive design verified
- [ ] Animations perform well
- [ ] No console errors in browser

### Performance

- [ ] Lighthouse audit run locally (Performance 80+)
- [ ] Images optimized and compressed
- [ ] No unnecessary dependencies
- [ ] Bundle size reasonable
- [ ] Lazy loading implemented for heavy components

### Accessibility

- [ ] Keyboard navigation works on all pages
- [ ] Focus indicators visible
- [ ] Alt text on all images
- [ ] Semantic HTML used
- [ ] Color contrast meets WCAG AA
- [ ] Screen reader tested (basic check)

### SEO

- [ ] Page titles are descriptive and unique
- [ ] Meta descriptions added
- [ ] Open Graph images set
- [ ] Sitemap generated
- [ ] Robots.txt configured
- [ ] Structured data implemented

### Domain & DNS

- [ ] Domain kazekeza.com purchased and active
- [ ] Access to Porkbun DNS management
- [ ] DNS records ready to update
- [ ] Understand DNS propagation may take time

### Documentation

- [ ] README.md updated with project info
- [ ] Deployment guide reviewed
- [ ] Environment variables documented
- [ ] Build process documented
